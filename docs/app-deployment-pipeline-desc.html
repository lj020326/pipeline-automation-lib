Information for the Deployment Jobs pipeline framework is located at How to use the App Deployment Pipeline.

The current deployment environment configuration is defined in the yaml section here and information on how to deploy app components using the framework is here.

It is assumed that the yaml definition will be updated as versions are promoted through each environment. Therefore, it is best to leave the first parameter "ArtifactVersion" blank unless necessary to override the environment defaults. This also works to make the pipeline more efficient since input overrides should not be necessary unless an exception or edge case is desired for deployment.
